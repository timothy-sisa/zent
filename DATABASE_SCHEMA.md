# Database Schema

## Collection: users

| Field         | Type         | Required | Notes                                               |
|---------------|--------------|----------|-----------------------------------------------------|
| _id           | ObjectId     | Auto     | MongoDB generated ID                                |
| username      | String       | Yes      | Unique, 3–30 chars, alphanumeric                    |
| email         | String       | Yes      | Unique, lowercase                                   |
| password      | String       | Yes      | Stored as bcrypt hash                               |
| role          | String       | Yes      | "student" or "lecturer", defaults to "student"      |
| favourites    | ObjectId[]   | No       | Array of Resource IDs                               |
| createdAt     | Date         | Auto     | Mongoose timestamp                                  |
| updatedAt     | Date         | Auto     | Mongoose timestamp                                  |

---

## Collection: resources

| Field           | Type         | Required | Notes                                              |
|-----------------|--------------|----------|----------------------------------------------------|
| _id             | ObjectId     | Auto     | MongoDB generated ID                               |
| title           | String       | Yes      | Max 150 chars                                      |
| description     | String       | Yes      | Max 1000 chars                                     |
| category        | String       | Yes      | e.g. "Computer Science"                            |
| resourceType    | String       | Yes      | lecture_notes, past_paper, study_guide, other      |
| filePath        | String       | Yes      | GridFS ObjectId of the uploaded file               |
| originalFileName| String       | Yes      | Filename as uploaded by the user                   |
| uploadedBy      | ObjectId     | Yes      | Reference to User                                  |
| ratings         | Array        | No       | Embedded rating sub-documents                      |
| ratings.user    | ObjectId     | Yes      | Reference to User who rated                        |
| ratings.value   | Number       | Yes      | Integer 1–5                                        |
| averageRating   | Number       | Auto     | Calculated from ratings array                      |
| viewCount       | Number       | Auto     | Incremented on each view, defaults to 0            |
| createdAt       | Date         | Auto     | Mongoose timestamp                                 |
| updatedAt       | Date         | Auto     | Mongoose timestamp                                 |

---

## Collection: comments

| Field      | Type      | Required | Notes                                 |
|------------|-----------|----------|---------------------------------------|
| _id        | ObjectId  | Auto     | MongoDB generated ID                  |
| resource   | ObjectId  | Yes      | Reference to Resource                 |
| author     | ObjectId  | Yes      | Reference to User                     |
| body       | String    | Yes      | 1–1000 chars                          |
| createdAt  | Date      | Auto     | Mongoose timestamp                    |
| updatedAt  | Date      | Auto     | Mongoose timestamp                    |

---

## Collection: sessions

| Field      | Type    | Notes                                              |
|------------|---------|----------------------------------------------------|
| _id        | String  | Session ID generated by express-session            |
| session    | Object  | Stores userId and recentlyViewed array             |
| expires    | Date    | TTL of 14 days — MongoDB auto-deletes expired      |

---

## Collection: uploads.files + uploads.chunks (GridFS)

These two collections are created automatically by MongoDB GridFS when a file is uploaded.

| Collection      | Purpose                                               |
|-----------------|------------------------------------------------------|
| uploads.files   | Stores file metadata — filename, size, content type, upload date |
| uploads.chunks  | Stores the actual file data split into 255KB chunks  |
